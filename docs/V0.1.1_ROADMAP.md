# v0.1.x Roadmap — Installation & Auto-Updates

## Status: Complete (v0.1.4)

DMG distribution and auto-updates via GitHub Releases are working. Unsigned builds require `xattr -cr` on first launch. Code signing deferred until Apple Developer enrollment.

---

## What's done

### DMG Distribution
- Bundle targets configured (`"all"` + `createUpdaterArtifacts: true`)
- GitHub Actions release workflow (`.github/workflows/release.yml`): builds on `v*` tag push, Apple Silicon only
- GitHub Actions CI workflow (`.github/workflows/ci.yml`): runs tests + clippy + tsc on push/PR
- macOS minimum version set to 12.0

### Auto-Updates
- `tauri-plugin-updater` integrated (Rust + frontend)
- Updater key pair generated, private key in GitHub Secrets
- Public key in `tauri.conf.json`, endpoint points to GitHub Releases `latest.json`
- Frontend: startup update check, progress bar with percentage, error display
- "Relaunch" prompt after download (auto-restart blocked for unsigned apps)

### Model Improvements (v0.1.2+)
- Gemma 4 E2B added to curated catalog (5.1B params, ~3 GB)
- Simplified setup wizard: one-click model cards, no quant dropdown
- HuggingFace model browser in Settings: search or paste repo URL, browse .gguf files
- Human-readable quant labels across all UI (Q4_K_M → "4-bit")

## When Apple Developer account is ready

Three steps to enable code signing + notarization:

1. **Add GitHub Secrets:**
   - `APPLE_CERTIFICATE` — base64 of .p12 Developer ID cert
   - `APPLE_CERTIFICATE_PASSWORD`
   - `APPLE_ID` — Apple ID email
   - `APPLE_PASSWORD` — app-specific password
   - `APPLE_TEAM_ID` — 10-char team ID

2. **Update `tauri.conf.json`:**
   ```json
   "macOS": {
     "signingIdentity": "Developer ID Application: Your Name (TEAMID)",
     "minimumSystemVersion": "12.0"
   }
   ```

3. **Add certificate import step to `release.yml`** (was previously removed)

Benefits: no `xattr` workaround, no Gatekeeper warnings, auto-restart after update works.
