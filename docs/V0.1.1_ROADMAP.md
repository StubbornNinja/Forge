## Goal

Get Forge installable from a `.dmg` on macOS with auto-update support via GitHub Releases. The "it just works" experience: download, drag to Applications, open, use. When a new version drops, the app notifies and updates in-place.

## Prerequisites

- GitHub repository (public or private)
- Apple Developer account (for code signing + notarization)
- GitHub Actions for CI/CD

---

## Phase 1: DMG Installation

### 1.1 Code Signing & Notarization

- Enroll in Apple Developer Program (if not already)
- Generate Developer ID Application certificate
- Set up `tauri.conf.json` signing config:
  ```json
  "bundle": {
    "macOS": {
      "signingIdentity": "Developer ID Application: Your Name (TEAMID)",
      "notarization": {
        "teamId": "TEAMID"
      }
    }
  }
  ```
- Store credentials as GitHub Secrets: `APPLE_CERTIFICATE`, `APPLE_CERTIFICATE_PASSWORD`, `APPLE_ID`, `APPLE_PASSWORD`, `APPLE_TEAM_ID`

### 1.2 Build Configuration

- Configure `tauri.conf.json` bundle targets for macOS:
  ```json
  "bundle": {
    "targets": ["dmg", "updater"],
    "macOS": {
      "minimumSystemVersion": "12.0"
    }
  }
  ```
- Tauri v2 generates `.dmg` with drag-to-Applications layout automatically
- Test `cargo tauri build` locally first to verify DMG output

### 1.3 GitHub Actions CI

- Create `.github/workflows/release.yml`:
  - Trigger: push tag `v*` (e.g. `v0.1.1`)
  - Matrix: macOS (arm64 + x64), later Linux
  - Steps: checkout â†’ setup Rust/Node â†’ install deps â†’ `cargo tauri build` â†’ sign â†’ notarize â†’ upload artifacts
- Create `.github/workflows/ci.yml`:
  - Trigger: push to `main`, PRs
  - Steps: `cargo test`, `cargo clippy`, `npx tsc --noEmit`

### 1.4 GitHub Release

- Tag `v0.1.0` on current main as the baseline
- Build and upload the DMG manually or via CI
- Verify: download DMG â†’ mount â†’ drag to Applications â†’ launch â†’ works

## Phase 2: Auto-Updates

### 2.1 Tauri Updater Plugin

- Install `@tauri-apps/plugin-updater` (frontend) and `tauri-plugin-updater` (Rust)
- Add to `tauri.conf.json`:
  ```json
  "plugins": {
    "updater": {
      "endpoints": [
        "https://github.com/OWNER/REPO/releases/latest/download/latest.json"
      ],
      "pubkey": "YOUR_PUBLIC_KEY"
    }
  }
  ```
- Generate updater key pair: `cargo tauri signer generate -w ~/.tauri/forge.key`

### 2.2 Update Manifest

- CI generates `latest.json` during release build (Tauri does this automatically with the `updater` bundle target)
- Format:
  ```json
  {
    "version": "0.1.1",
    "notes": "Bug fixes and improvements",
    "pub_date": "2026-04-05T00:00:00Z",
    "platforms": {
      "darwin-aarch64": {
        "signature": "...",
        "url": "https://github.com/.../Forge_0.1.1_aarch64.app.tar.gz"
      }
    }
  }
  ```

### 2.3 Frontend Integration

- On startup, call updater plugin's `check()` function
- If update available, push notification via existing `notificationStore`:
  ```typescript
  addNotification({
    id: 'forge-update',
    message: `Forge v${latest} available`,
    action: { label: 'Update', handler: () => installUpdate() },
    dismissable: true,
  });
  ```
- `installUpdate()` calls the plugin's `install()` â†’ downloads, verifies signature, restarts app
- Show download progress in the notification or a modal

### 2.4 Version Bump Workflow

- When releasing: bump version in `tauri.conf.json` and `Cargo.toml`
- Tag the commit: `git tag v0.1.1`
- Push: `git push origin v0.1.1`
- CI builds, signs, notarizes, uploads to GitHub Releases with `latest.json`
- Users get notified on next app launch
